# Licensing and Attribution

This repository aggregates original work, adapted material, and bundled skill packs that may carry different provenance or licensing terms. Because of that, the repository should not be treated as if every directory is covered by one blanket license unless a specific file says so.

## How to Read Licensing in This Repo

- If a folder contains `LICENSE`, `LICENSE.txt`, or equivalent attribution files, treat that folder-level notice as authoritative for that content.
- If a skill pack references an upstream project, preserve that attribution when reusing or redistributing it.
- If no explicit license is present for a directory or file, do not assume open reuse rights.

## Original Repository Material

Unless noted otherwise, the following items should be treated as original repository documentation or project structure material:

- root documentation files such as `README.md`, `CONTRIBUTING.md`, `SECURITY.md`, and this file
- repository organization choices such as ignore rules and publish-safe structure
- the portfolio site content under `portfolio/`, unless another embedded asset states otherwise

## Contributor Expectations

When adding or modifying content:

- keep existing license notices intact
- do not remove upstream attribution
- add attribution or license text when importing third-party assets
- avoid relicensing third-party material without permission

## When in Doubt

If you want to reuse a large portion of this repository and the licensing status is unclear, contact the repository owner first instead of assuming permission.
