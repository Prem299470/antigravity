# Security Policy

This repository includes cybersecurity learning material, automation assets, and a public portfolio. If you discover a security issue in tracked code, documentation, or published configuration, please report it responsibly instead of disclosing it immediately in public.

## How to Report

- Email: `kprv991@gmail.com`
- GitHub: [github.com/Prem299470](https://github.com/Prem299470)

Please include:

- a short description of the issue
- the affected file or directory
- reproduction steps or evidence, if applicable
- the potential impact

## Response Expectations

- Good-faith reports will be reviewed as quickly as possible.
- Sensitive issues should be kept private until they are assessed and, where needed, remediated.

## Scope

This policy applies to the tracked contents of this repository, including:

- root documentation and configuration
- the static portfolio in `portfolio/`
- scripts or helper assets committed under `skills/`

## Safe Use Notice

Some repository content discusses offensive security techniques, vulnerability assessment, or pentesting workflows. Use those materials only in authorized environments and in accordance with applicable law, platform policy, and professional ethics.

## Publish Hygiene

If you notice accidentally committed secrets, private paths, personal data, or local runtime artifacts, report that immediately so it can be removed and rotated where necessary.
